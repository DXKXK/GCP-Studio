## 开发规范

##### 命名方式：

​	命名规则：尽量使用对应英文

​	全小写：包名，接口名

​	大驼峰命名：类名

​	小驼峰命名：变量名，函数名，方法名

**响应规则：**

​	关于接口响应数据的结构统一为如下结构

```
{
    "code": "响应状态码",
    "msg": "响应信息简述",
    "data": "响应数据，如果无需返回数据则设为：NULL或是{}"
}
```

##### 文件位置：

```
	Request包：负责接口相关的代码
		相同类型的接口写在同一java文件中
		（如：测试用的接口在RequestTest类中，用户相关(注册,登录,...)在User类中）

	Tools包：负责其他逻辑处理相关代码

	Admin包：管理后台操作专用，包含接口和逻辑处理相关
	
	如有其他自行添加。。。。。。。
```

##### 注释规则：

​	尽量在每一个接口前需要写清楚作用。

##### 功能实现：

​	允许使用AI，**但是不能直接照搬AI给出的代码或是由AI直接修改代码文件（避免AI在一些奇奇怪怪的位置添加一些奇奇怪怪的语句），**该项目开发的目的之一也是为了锻炼编码能力，请使用对AI给出的代码加以理解后的代码

### 功能实现

- **安全相关**

  - [ ] 用户密码加密处理

    - ```
      #### 密码加密

      前端传递后的密码应当是经过哈希算法加密过的，后端需要进一步加密
      后端采用bcrypt加密，加密因子应大于15
      ```

  - [x] 生成TOKEN（JWT令牌）

  - ```
    #### 令牌荷载
      "email": "用户邮箱"
      "role": 用户权限（普通用户，工作室成员，管理员）
      "exp": "令牌过期时间"（自创建起48小时）
      "iat": "令牌签发时间"
    ```

  - [x] 路径拦截

  - ```
    根据登录状态定向资源页面路径，防止出现绕过登录直接访问管理后台页面的情况
    ```

- **数据库相关**

  - [ ] 建立用户表（存贮用户信息，采用MySQL数据库）

  - ```
    #### 数据库字段设计
    	id：自增主键
    	username：用户名称
    	email：用户邮箱（唯一标识）
    	password：用户密码（前端哈希加密和后端bcrypt加密后的字符串）
    	token：用户令牌，校验时需要验证是否过期，可被覆盖，可为null
    	time: 该用户信息建立时间
    	avatar：用户投头像路径
    ```

  - [ ] 建立用户表（记录用户图形验证码的正确内容，采用Redis数据库）

  - ```
    前端发起获取图形验证码请求，根据用户邮箱写入响应的图形验证码的正确内容
    先进行验证码比对，再进行密码比对
    ```


  - [ ] 建立用户表（记录用户邮箱验证码的正确内容，采用Redis数据库）

  - ```
    前端发起发送邮箱验证码请求，根据用户输入的邮箱验证码和服务器发送的验证码进行验证
    ```

##### **admin包（Admin，AdminSession，SessionIntercept），管理后台操作专用**

```
###### 	Admin
	这里配置根路径重定向，将资源页面定向到"AdminLogin/index.html"
	
###### 	AdminSession
	这里配置登录会话相关的接口（登录，退出登录）
	
######  SessionIntercept
	这里配置了路径拦截器，根据登录状态定向资源页面路径，防止出现绕过登录直接访问管理后台页面的情况
```

##### **tools包（WebConfig），逻辑处理相关**

##### tools包（CaptchaImage），图片验证码生成专用

```
###### 	CaptchaImage
	该类是用于处理图片验证码生成专用相关，内部提供了两个方法
	1：createCaptcha()，
		作用：生成验证码图片，并放置的对应位置
		传入值：email(用户邮箱)
		返回：code[String],path[String](验证码答案，验证码路径)
	2：clearCaptcha（），
		作用：清除对应用户文件夹下的验证码图片
		传入值：email(用户邮箱)
		返回：已清除:true/未清除:false
		
	上述方法 不是静态方法，而是实例方法。调用时需要通过 CaptchaImage 的实例进行访问
	
	待实现功能，使用内存数据库Redis对验证答案进行缓存，并定期清理过期的验证码
```

##### tools包（JwtToken），令牌生成相关

```
######	JwtToken
	该类是用于处理Token相关的，内部提供了三个方法
	1：createToken()，
		作用：生成Token
		传入值：email(用户邮箱)，role(用户权限)
		返回：Token[Strig]
   	2：checkExp()
   		作用：校验Token的时效性
   		传入值：token(用户令牌)
   		返回：未过期:true/过期:false
   	3：checkRole()
   		作用：校验Token身份
   		传入值：token(用户令牌)，role(需要进行校验的身份)
   		返回：是:true/否:false
   		
	上述方法 不是静态方法，而是实例方法。调用时需要通过 JwtToken 的实例进行访问。
```
