# 匠心客工作室官网前端页面开发：

## 项目简述：

开发方式：多人按模块开发，不同成员负责不同模块进行开发

```
- 框架：Vue3 + Vite
- 语言：JavaScript
- 包管理：npm/yarn
- UI框架：Element Plus
- 路由：Vue Router 4.5.1
- HTTP库：Axios
```



## 开发规范：

##### 命名规则：

​	尽量使用对应英文，避免出现（aaa , bbb，……之类的，测试用的除外）
	"*.vue"文件的命名采用大驼峰的命名方式，

##### 文件放置规则：

```
###### 	页面位置：
		统一放在“./gcp_studio_web/src/views”，不要单独建立一个“*.vue”文件，需要放在同名的目录下

###### 	组件位置：
		对与整个项目使用的组件，放在“./gcp_studio_web/src/components”，页面专用组件则放在对应页面下的“components”目录

###### 	图片位置：
		统一放在“./gcp_studio_web/src/assets/images”下，注意需要在此处建立与页面同名的目录，图片要放在这个目录下。(如：index.vue中使用到的图片放在"./gcp_studio_web/src/assets/images/index"下)

###### 	页面样式：
		鉴于页面样式代码可能会比较多，因此需要将样式代码独立开来，和页面放在同一个目录，统一命名为"style"，使用css或是scss开发成员自行选择，如果样式不多可以直接写在vue文件中

###### 	脚本代码（JS）：
		使用：组合式 API + <script setup> 语法，可独立开来也可以写在vue文件中，统一命名为"script.js"
```



##### 注释规则：

​	尽量在每一个方法名中需要写清楚作用。

##### 编写规则：

​	在确认功能开发完成之后，合并分支之前请注释掉或删掉所有非必要的调试输出语句

##### 功能实现：

​	允许使用AI，**但是不能直接照搬AI给出的代码或是由AI直接修改代码文件（避免AI在一些奇奇怪怪的位置添加一些奇奇怪怪的语句），**该项目开发的目的之一也是为了锻炼编码能力，请使用对AI给出的代码加以理解后的代码

**主页面（Index.vue），参考："https://pvp.qq.com/","https://www.mihoyo.com/"**

```
因此将子页面设置为组件的形式使用，这些子页面的性质是"Index.vue"，专用组件
因此如果需要添加新的子页面需要按照一下步骤设置
    1：在"Index/components"目录下添加"PageX.vue"文件(X表示页面序号)
    2：在"Index/script.js"中的"page"变量添加页面基本信息
    3：在script代码块中引入对应的页面的vue文件
    4：在<template>代码块中参照已有的页面组件形式，应用对应页面的的组件，并调整顺序
    
同时对应的页面的vue文件中也需要设置
    1：在<template>代码块下最外层有且仅能有一个<div>盒子，且包含类名"content-box"
    2：给最外层盒子"content-box"设置与在"Index/script.js"中添加的相同的id
```



**网路请求（"api/request.js"）**

```
1：在该文件中已经封装好一个请求方法，可供网络请求调用
    1)在需要网络请求的vue中的script代码块中引入
    ——示例：import { scroll } from 'script.js的相对路径'）

    2)在需要的地方异步调用"request()"即可
    ——示例：
    	const req = async() => {
    		// res中就有请求到的数据
    		// 请求链接：项目配置了代理Host，因此请求链接不需要写完整链接，例如'/api/getlist'
    		// 请求方法：'GET','POST'等方法
    		// 请求参数：如果是POST请求，则需要传入一个对象
    		// 请求参数：如果是GRT请求，则需要传入字符串
    		// 请求头：接口设置有安全验证，参照API接口文档配置即可
    		res = await request('请求链接', '请求方法' , '请求参数', '请求头')
    	};
2：也可以自己写一个请求方法导出，注意网路请求相关的方法必须写在"script.js"中方便后续网络请求报错排查

```

**项目配置（"vite.config.js"）**

此处随着后续项目的变化，由项目负责人修改

```
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// https://vite.dev/config/
export default defineConfig({
  plugins: [vue()],
  server:{
    host: '0.0.0.0',		// 允许同一局域网设备访问开发服务器
    port: 3000,				// 开发服务器端口设置，建议项目打包上线该为其他不易冲突的端口
    proxy: {				// 请求代理
      '/api': {
        target: 'http://127.0.0.1:8080', // 后端服务器地址，项目打包上线需要改为实际的服务器地址
        changeOrigin: true,		// 允许请求头中的Host改为后端服务器地址
      },
    }
  }
})

```
